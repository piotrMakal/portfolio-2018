Maya Wire Plugin

Year: 2017 - Present
Fields: 3D Modeling, Mathematics, Programming, Tools Development
Used: Autodesk Maya, Autodesk Maya API, Python
Description: Maya Wire Plugin enables fast and easy creation of wires, cables, pipes, ropes, ribbons and other wire-like shapes.

Why Bother?

Autodesk Maya has its own ways for wire creation, so why creating separate tool for this? In my opinion there are exactly 3 reasons: (1) it takes too much time to create a simple cable, (2) final result is hard to maintain/modify later on, (3) the algorithms Maya provides leave a lot to be desired.
I was especially affected by this when working on a game, where I had to model a tons of cables. That's where the original idea came to my mind.

License: CC BY-SA 4.0

With plugin comes a new node (wireMeshCreator) that is responsible for all the calculations and outputing correct wire mesh data. This allows modifications to geometry at any point in time, even after saving and reopening Maya scene. Attributes of the node can be mixed together giving user the flexebility of achiving different shapes, from real-world: cables, pipes, ribbon and ropes, to more abstract ones.

My First Real-time Renderer

License: CC BY-SA 4.0

In late 2015, after a few months of learning OpenGL, I wrote my very first simple real-time renderer in C++, with the help of GLEW, SDL, GLM and boost. Renderer implements most basic features like: Lambert and Phong shaders, textures with alphas, normal maps, different types of light (ambient, point, spot and directional) with depth map shadows, orbiting camera and more.

Toltec

License: CC BY-SA 4.0

Toltec is my project for learning purposes that I was working on in 2016 and 2017 in my spare time. The goal was to create multi-context, multi-threaded 3D modeling application with a support for different rendering systems (one of which was inspired by Autodesk Maya Viewport 2.0). For this project I used C++ with Qt, glbinding and GLM. Currently this project is suspended/cancelled due to its complexity vs. little time I could spend on it.

Custom Axis Orientation in Degrees (Maya Script)

License: CC BY-SA 4.0

Python script for Autodesk Maya that I wrote in 2013. Back then Maya users have to type custom axis for move, rotate and scale tool using radians, which was wierd. This script became somewhat obsolete in 2015 when Autodesk Maya finally switched to degrees in that subject, although script may be still usefull in certain situations.
